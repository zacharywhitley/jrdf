Differences between SPARQL and DRQL

Semantics

  • Joins - SPARQL allows nulls to be treated as values and can be accepted during join.
  • Compatbility - SPARQL allow the relations to be the same type. For example:
    • {{ (?x = 2, ?y = 3) }} \ {{ (?y = 3) }} is {} in SPARQL
    • {{ (?x = 2, ?y = 3) }} - {{ (?y = 3) }} is {{ (?x = 2, ?y = 3) }}

Syntax

  • SELECT {} instead of SELECT *
  • SELECT { a, b, c} instead of SELECT ?a ?b ?c
  • JOIN instead of "."
